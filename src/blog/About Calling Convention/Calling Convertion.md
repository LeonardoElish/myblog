
## 一、什么是调用约定？

**调用约定**决定了函数参数的传递方式、调用和返回时栈的管理方式，以及函数名的修饰规则。不同调用约定在跨平台、与操作系统或第三方库交互时非常重要。

## 二、常见调用约定

### 1. __cdecl

- **全称**：C Declaration
- **参数传递**：从右到左入栈。
- **栈清理者**：调用者负责在调用函数后清理栈（例如参数）。
- **函数名修饰**：下划线前缀，例如 `_FunctionName`
- **优点**：参数个数可变（如 `printf`），是 C/C++ 默认调用约定。
- **用途**：普通 C/C++ 函数，尤其是可变参数函数。

**示例**：
```cpp
int __cdecl sum(int a, int b);
```

### 2. __stdcall

- **参数传递**：从右到左入栈。
- **栈清理者**：被调用函数负责清理栈。
- **函数名修饰**：前有下划线，后面加 `@` 和参数字节数（如 `_FunctionName@8`）
- **优点**：调用者代码更简洁，常用于 WinAPI。
- **用途**：Windows API、大多数系统 DLL 导出的函数。

**示例**：
```cpp
int __stdcall sum(int a, int b);
```
**WINAPI** 就是 `__stdcall` 的宏定义。

### 3. __fastcall

- **参数传递**：通过寄存器（如 ECX、EDX）传前两个参数，剩余参数仍从右到左压栈。
- **栈清理者**：被调用函数负责清理栈。
- **函数名修饰**：前有 `@`，后面加参数字节数（如 `@FunctionName@8`）
- **优点**：速度更快，减少内存访问。
- **用途**：需要高性能的函数（但兼容性差，不常用于跨模块接口）。

**示例**：
```cpp
int __fastcall sum(int a, int b);
```

### 4. __thiscall

- **参数传递**：用于 C++ 类的成员函数，`this` 指针通过 ECX 传递，其他参数从右到左压栈。
- **栈清理者**：被调用函数负责清理栈。
- **函数名修饰**：C++ 的名字修饰规则（mangling）。
- **用途**：C++ 类成员函数（只在 C++ 下有效）。

**示例**：
```cpp
class MyClass {
    int __thiscall sum(int a, int b);
};
```

### 5. __vectorcall（现代 MSVC）

- **参数传递**：用向量寄存器传递参数（如 SIMD 类型），其余参数压栈。
- **优点**：用于高性能数值计算、多媒体等。
- **用途**：现代高性能代码（仅新 MSVC 支持）。

**示例**：
```cpp
void __vectorcall func(float a, float b, float c);
```

---

## 三、默认约定和宏

- Windows 下普通函数默认 `__cdecl`，WinAPI 用 `__stdcall`（即 `WINAPI` 宏）。
- `WINAPI` 等价于 `__stdcall`
- `CALLBACK` 通常也是 `__stdcall`

---

## 四、调用约定的影响

1. **跨 DLL/模块调用**：调用约定必须与被调用函数一致，否则会导致栈混乱、崩溃。
2. **参数可变性**：仅 `__cdecl` 支持可变参数（如 `printf`）。
3. **性能优化**：如 `__fastcall`、`__vectorcall` 使用寄存器加速。

## 五、表格汇总

| 调用约定     | 参数入栈顺序  | 清理者   | 可变参数 | 典型应用       | 兼容性 |
| ------------ | ------------- | -------- | -------- | -------------- | ------ |
| __cdecl      | 右到左        | 调用者   | 支持     | C/C++ 普通函数 | 高     |
| __stdcall    | 右到左        | 被调用者 | 不支持   | Windows API    | 高     |
| __fastcall   | 寄存器+右到左 | 被调用者 | 不支持   | 性能优化       | 一般   |
| __thiscall   | ECX+右到左    | 被调用者 | 不支持   | C++成员函数    | 仅C++  |
| __vectorcall | 向量寄存器    | 被调用者 | 不支持   | 高性能代码     | 新MSVC |
