使用预编译头文件（PCH, Precompiled Header）是加快C++项目编译速度的常见方法。
下面详细介绍如何正确使用预编译头：
## 预编译头的基本原理

预编译头文件通常用于存放不经常改动的大型头文件（如 STL、Windows.h、第三方库等），编译器会将这些头文件预先编译为一个特殊的二进制文件（如 `pch.pch`），后续源文件只需引用该PCH文件即可，省去重复解析和编译的时间。

## VS2022 中的标准用法

### 1. 创建预编译头文件

假设你的项目名为 `MyProject`，按照如下步骤操作：
#### 步骤一：新建头文件
`pch.h`
```c++
///pch.h
#ifndef PCH_H
#define PCH_H

// 在此处包含要预编译的头文件
#include <iostream>
#include <vector>
```

#### 步骤二：新建源文件 `pch.cpp`

```c++
///pch.cpp
#include "pch.h"
```

该文件仅包含对 `pch.h` 的一次引用。


###  配置项目属性

1. **右键项目 → 属性（Properties）**
    
2. **导航到 C/C++ → 预编译头（Precompiled Headers）：**
    
    - 预编译头（Precompiled Header） = **使用 (/Yu)** （对于普通源文件）
    - 预编译头文件（Precompiled Header File）= `pch.h`
    - 预编译头输出文件（Precompiled Header Output File）= `$(IntDir)\pch.pch` （默认即可）
3. **仅对 `pch.cpp` 设为“创建 (/Yc)”**
    
    - 选中 `pch.cpp` 文件，在属性→C/C++→预编译头，将“预编译头”改为**创建 (/Yc)**
4. **所有需要PCH的源文件的最顶部都加上：**
        
    ```c
    #include "pch.h"
    ```

### 使用建议
- 只包含不常改动、通用的头文件。
- 如果某些文件不需要PCH，可以在属性中将“预编译头”设为**不使用**。
- 一旦 `pch.h` 更改，所有引用它的源文件都需要重新编译。
