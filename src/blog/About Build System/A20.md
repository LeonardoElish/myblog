# A20 Gate（A20 门）详解

本文介绍 A20 Gate 的背景、为什么会存在、它如何影响地址计算，以及常见的启用/禁用方法（含汇编与 C 示例）、在引导与切换到保护模式过程中的注意事项与调试技巧。

## 背景与历史

- 在早期的 x86（8086）架构中，CPU 使用 20 位地址线（A0..A19），有效物理地址范围为 0x00000 到 0xFFFFF（1 MiB）。地址计算在实模式下由段寄存器左移 4 加上偏移完成，但结果只保留低 20 位。这会导致当地址超过 1 MiB 时“回绕”（wrap around）回到 0x00000（例如 0xFFFF0 + 0x10 = 0x00000）。
- 随着 IBM PC AT（80286/80386）等出现并引入更多物理地址线（和保护模式），许多旧的 DOS 程序、游戏或 BIOS 依赖这种 20 位回绕行为来访问特定内存位置或做遮盖运算。为了保持向后兼容，IBM 在 AT 机器上实现了一个名为 A20 Gate 的机制，用于临时禁止或允许 A20 地址线（第 21 位，A20，注意编号从 A0 开始），以便控制是否发生回绕。

简言之：
- A20 line = CPU 地址线编号为 20（从 A0 起算，即第 21 条地址线），控制是否允许访问超过 1 MiB 的地址空间。
- A20 Gate 关闭时，地址会按 20 位回绕；A20 Gate 打开时，地址使用更高位，允许访问 1 MiB 以上的物理内存。

## 为什么内核/引导器要关心 A20

- 在现代内核或引导加载器中，通常希望 A20 被打开（enabled），以便能正确访问 1MiB 以上的内存（例如加载内核、使用 BSS、页表等）。
- 在切换到保护模式之前（或在 early boot 初始化内存前），需要确保 A20 已开启，否则在开启分页或访问高内存时会造成数据错乱或内核无法正确定位内存。

## 常见启用/禁用 A20 的方法

下面列出常见且广泛使用的三种方法：

1. 通过端口 0x92（System Control Port A，快速方法）
2. 通过键盘控制器（8042）命令：读取输出端口、修改 A20 位、写回（兼容性更好但慢）
3. 通过 BIOS/固件提供的服务（部分 BIOS/UEFI 提供开/关 A20 的接口）

下面逐一说明并给出示例代码。

### 方法 1：端口 0x92（Fast A20）

许多现代 PC 支持通过主板的系统控制端口 0x92 来快速启用 A20。该端口的位 1（mask 0x02）通常用作 A20 的开关。优点是简单且不依赖 8042 键盘控制器的状态；缺点是并非所有旧硬件都支持该端口（但现代 PC/虚拟机普遍支持）。

示例（汇编）：

```asm
    in al, 0x92
    or al, 0x02    ; 设置 bit1
    out 0x92, al
```

示例（C，使用 `outb`/`inb`）：

```c
#include <stdint.h>
static inline uint8_t inb(uint16_t port);
static inline void outb(uint16_t port, uint8_t v);

void enable_a20_port92(void) {
    uint8_t v = inb(0x92);
    v |= 0x02;
    outb(0x92, v);
}

void disable_a20_port92(void) {
    uint8_t v = inb(0x92);
    v &= ~0x02;
    outb(0x92, v);
}
```

在很多引导器中，优先尝试此方法，如果不可用再降级到 8042 方法。

### 方法 2：通过 8042 键盘控制器（兼容方法）

键盘控制器（8042）在早期 PC 中负责 A20 管理；它通过“输出端口”（output port）的一位来控制 A20。启用/禁用通常需要：

- 使用命令 0xD0 读取控制器输出寄存器（然后从端口 0x60 读回它的值）
- 修改读回的输出值的 A20 位（通常是 bit 1），然后用命令 0xD1 将修改后的值写回（写入到端口 0x60）。

注意：8042 交互需要在发送命令/数据前检查控制器状态寄存器（端口 0x64）的输入/输出缓冲区标志，以确保控制器准备好读取/写入。

伪代码（概念性步骤）：

```c
// 等待输入缓冲区为空
wait_for_input_buffer_empty();      // poll (inb(0x64) & 2) == 0
outb(0x64, 0xD0);                   // 读输出端口命令
wait_for_output_buffer_full();      // poll (inb(0x64) & 1) == 1
uint8_t outp = inb(0x60);           // 读取当前输出端口

outp |= 0x02;                       // 设置 A20 位

wait_for_input_buffer_empty();
outb(0x64, 0xD1);                   // 写输出端口命令
wait_for_input_buffer_empty();
outb(0x60, outp);                   // 将修改值写回
// 可选：等待并验证
```

这种方法对老旧系统兼容性最好，但要注意正确的等待（poll）循环和超时，否则在某些硬件上可能死锁。引导程序（如 GRUB）中常用此方式作为后备。

### 方法 3：使用 BIOS / 固件服务

部分 BIOS/固件提供了用于开启/关闭 A20 的中断接口或服务（例如早期 BIOS 的 INT 15h 子功能），你可以在 real-mode 下调用 BIOS 服务来完成。优点是实现简单且更为抽象；缺点是在某些现代 UEFI 或无 BIOS 的环境中不可用。

示例（伪）: 使用 BIOS 服务打开 A20（依赖 BIOS 实现）

```asm
    mov ax, 0x2401  ; BIOS 子功能（各 BIOS 实现不同，请参照具体 BIOS 手册）
    int 0x15
```

> 提示：不要倚赖单一 BIOS 中断编号——不同 BIOS 版本实现可能不同，实际使用时请查阅目标平台文档或使用更通用的方法（先尝试端口 0x92，再尝试 8042）。

## 在引导流程中的使用建议

- 在引导加载器（bootloader）里，尽早确保 A20 已打开：在加载内核到 1MiB 以上地址或设置分页前应启用 A20。
- 若你实现了多种方法：优先尝试 0x92（fast），失败后回落到 8042（兼容），最后尝试 BIOS 服务（如果在实模式并且 BIOS 支持）。
- 在切换到保护模式（或使用 32-bit 平坦模型）前确保 A20 已启用，否则段算术可能出现回绕导致内核定位/内存访问错误。

## 常见问题与调试技巧

- 问：如何验证 A20 是否已启用？
  - 简单测试：在实模式下写入地址 `0x000FFFF0` 处的字节（或检查通过段:offset 计算得到的地址是否与物理地址对应），然后访问 `0x00100000` 同样偏移处并判断是否回绕。也可以执行写入 0x00000000 后读回等技巧。
  - 更直接：使用小的测试代码在实模式下写入两处地址并比较是否相同（若相同说明回绕）。

- 问：为什么在某些虚拟机/模拟器中 A20 行为不同？
  - 虚拟化平台（QEMU/VirtualBox）通常默认启用 A20，或者提供配置项。某些老旧兼容选项可能改变默认行为。调试时请确认虚拟机设置或使用平台文档。

- 问：启用 A20 后是否会影响性能或缓存？
  - A20 本身只是控制地址线；启/关对性能影响可忽略。但在非常旧的平台上切换 A20 的实现可能需要等待或与键盘控制器交互，导致短暂延迟。现代系统通常在引导早期一次性启用。

## 代码示例：完整的 0x92 优先 / 8042 回退策略（伪 C）

```c
#include <stdint.h>
static inline uint8_t inb(uint16_t port);
static inline void outb(uint16_t port, uint8_t v);

int try_enable_a20_port92(void) {
    uint8_t v = inb(0x92);
    v |= 0x02;
    outb(0x92, v);
    // 简单等待/验证可选
    return 0; // 0 表示成功（可改为实际检测）
}

int try_enable_a20_8042(void) {
    // 伪代码：必须实现 wait_for_* 函数用于避免死锁
    wait_for_input_buffer_empty();
    outb(0x64, 0xD0);                // 读输出端口
    wait_for_output_buffer_full();
    uint8_t outp = inb(0x60);

    outp |= 0x02;                    // set A20 bit

    wait_for_input_buffer_empty();
    outb(0x64, 0xD1);                // 写输出端口
    wait_for_input_buffer_empty();
    outb(0x60, outp);
    // 验证可选
    return 0;
}

void enable_a20(void) {
    if (try_enable_a20_port92() == 0) return;
    if (try_enable_a20_8042() == 0) return;
    // 最后可尝试 BIOS INT15，如果处于实模式且 BIOS 支持
}
```

注：示例中的 `wait_for_input_buffer_empty()` / `wait_for_output_buffer_full()` 需要以超时保护实现以避免在无法响应的硬件上死锁。
