# 📝 什么是 SSL/TLS？

#### 历史背景

- **SSL (Secure Sockets Layer)**
  - 1994年：Netscape 开发 SSL 1.0（因安全问题从未公开发布）
  - 1995年：SSL 2.0 发布（存在严重安全漏洞）
  - 1996年：SSL 3.0 发布（修复了许多问题）
- **TLS (Transport Layer Security)**
  - 1999年：TLS 1.0 发布（基于 SSL 3.0）
  - 2006年：TLS 1.1
  - 2008年：TLS 1.2
  - 2018年：TLS 1.3（当前最新、最安全版本）

#### 定义

SSL/TLS 是一种**加密通信协议**，在客户端和服务器之间创建安全通道，确保数据传输的机密性、完整性和真实性。

### 二、SSL/TLS 的三大核心功能

#### 1. 🔐 加密（Encryption）

**对称加密 + 非对称加密**

- **非对称加密**（握手阶段）
  - 使用公钥和私钥对
  - 服务器用私钥解密，客户端用公钥加密
  - 算法：RSA、ECC（椭圆曲线加密）
- **对称加密**（数据传输阶段）
  - 使用相同的密钥加密和解密
  - 速度更快，适合大量数据
  - 算法：AES、ChaCha20

#### 2. 🛡️ 身份验证（Authentication）

- **SSL 证书**验证服务器身份
- 防止中间人攻击（MITM）
- 可选的客户端证书验证

#### 3. ✅ 数据完整性（Integrity）

- 使用 **消息认证码（MAC）** 或 **AEAD**
- 检测数据是否被篡改
- 确保数据在传输过程中的完整性

### 三、SSL/TLS 握手过程（Handshake）

```
客户端                                  服务器
  |                                      |
  |------- 1. Client Hello ------------->|
  |    (支持的加密套件、TLS版本)           |
  |                                      |
  |<------ 2. Server Hello ---------------|
  |    (选择的加密套件、证书)              |
  |                                      |
  |------- 3. 验证证书 ------------------>|
  |    (检查证书有效性)                   |
  |                                      |
  |------- 4. 密钥交换 ------------------>|
  |    (生成会话密钥)                     |
  |                                      |
  |<------ 5. Finished -------------------|
  |                                      |
  |====== 加密通信开始 ===================|
```

#### 详细步骤：

1. **Client Hello**
   - 客户端发送支持的 TLS 版本
   - 支持的加密套件列表
   - 随机数（用于后续密钥生成）
2. **Server Hello**
   - 服务器选择 TLS 版本和加密套件
   - 发送 SSL 证书（包含公钥）
   - 发送服务器随机数
3. **证书验证**
   - 客户端验证证书的有效性
   - 检查证书是否由受信任的 CA 签发
   - 检查证书是否过期
   - 验证域名是否匹配
4. **密钥交换**
   - 客户端生成"预主密钥"
   - 使用服务器公钥加密后发送
   - 双方使用随机数和预主密钥生成"会话密钥"
5. **握手完成**
   - 双方确认握手成功
   - 开始使用对称加密通信

### 四、SSL 证书详解

#### 证书类型

1. **DV（Domain Validation）域名验证证书**
   - 验证级别：低
   - 只验证域名所有权
   - 签发速度：快（几分钟到几小时）
   - 适用：个人网站、博客
2. **OV（Organization Validation）组织验证证书**
   - 验证级别：中
   - 验证域名 + 组织身份
   - 签发速度：1-3天
   - 适用：企业网站
3. **EV（Extended Validation）扩展验证证书**
   - 验证级别：高
   - 严格验证组织法律实体
   - 浏览器地址栏显示公司名称（旧版浏览器）
   - 签发速度：1-2周
   - 适用：金融、电商等高安全需求网站

#### 免费 vs 付费证书

**免费证书（如 Let's Encrypt）**

- ✅ 完全免费
- ✅ 自动化签发和续期
- ✅ 加密强度相同
- ❌ 只有 DV 类型
- ❌ 有效期短（90天）

**付费证书**

- ✅ 提供 OV/EV 选项
- ✅ 更长有效期（1年）
- ✅ 技术支持
- ✅ 保险赔偿
- ❌ 需要费用

### 五、HTTPS 的工作原理

Code

```
HTTP  + SSL/TLS = HTTPS
明文传输 + 加密层 = 安全传输
```

#### HTTP vs HTTPS

| 特性   | HTTP     | HTTPS               |
| ------ | -------- | ------------------- |
| 端口   | 80       | 443                 |
| 安全性 | 明文传输 | 加密传输            |
| 速度   | 稍快     | 稍慢（加密开销）    |
| SEO    | 较低     | 较高（Google 优先） |
| 证书   | 不需要   | 需要 SSL 证书       |
