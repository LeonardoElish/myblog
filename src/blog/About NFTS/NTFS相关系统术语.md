# 簇

簇(cluster)（或分配单元）是可用于存储文件的最小磁盘空间单位。Windows Server 2003 使用的所有文件系统都基于簇大小来组织硬盘，簇大小由簇包含的扇区数（硬盘上的存储单元）决定。例如，在采用 512 字节扇区的磁盘上，一个 512 字节的簇包含一个扇区，而一个 4 千字节 (KB) 的簇包含八个扇区。

# 簇序列

NTFS 卷上的簇从分区开头开始按顺序编号为逻辑簇号。NTFS 使用称为主文件表 (MFT) 的记录来存储文件系统中的所有对象，其结构类似于数据库。

在 NTFS 卷上，簇从扇区 0 开始；因此，每个簇都沿簇边界对齐。连续的簇用于文件存储，从而加快文件处理速度。

# 簇大小

由于 NTFS 根据卷的大小使用不同的簇大小，因此每个文件系统都有其最大支持的簇数。簇越小，磁盘存储信息的效率就越高，因为簇内未使用的空间不能被其他文件占用。文件系统支持的簇越多，使用该文件系统可以创建和格式化的卷就越大。NTFS 使用较小的簇大小，这使其成为一种更高效的文件组织结构。

# 逻辑簇号（LCN）

LCN是对卷的第一个簇到最后一个簇进行编号，只要知道LCN号和簇的大小以及NTFS卷在物理磁盘中的起始扇区就可以对簇进行定位，而这些信息在NTFS卷的引导扇区中可以找到，在系统底层也是用这种方法对文件的簇进行定位的。找到簇在磁盘中的物理位置的计算公式是：
簇的起始绝对扇区号 = 每簇扇区数*簇号+卷的隐含扇区数（卷之前的扇区总数）

# 虚拟簇号（VCN）

虚拟簇号则是将特定文件的簇从头到尾进行编号，这样做的原因是方便系统对文件中的数据进行引用，VCN并不要求在物理上是连续的，要确定VCN的磁盘上的定位需先将其转换为LCN。
