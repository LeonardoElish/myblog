# Observer

# 传统Observer设计模式的缺陷

在传统的面对对象(OO)观察者模式存在一些固有的缺陷,这些缺陷在c++中尤其明显

## ⚠️ 传统观察者模式的缺陷

### **过度耦合 (Strong Coupling):**

- 观察者模式要求观察者（Observer）必须是**基类**的派生类，以实现统一的 `Update` 接口。
- 这种继承关系导致观察者和主题（Subject）之间存在**强类型耦合**。
- 它限制了观察者类成员的**名字**、**参数**、**函数返回类型**，甚至**成员函数数量**，因为它们都必须符合基类 `Observer` 的定义。

## 多重继承的难题(Multiple Inheritance Issue):

- 如果一个类 `Foo` 需要观察**两种不同类型**的事件（例如，同时观察温度和时间），它就需要**多重继承**两个不同的 `Observer` 基类。

- 当需要观察**重复或相似**的事件时（例如，既想观察 1 秒一次的心跳，又想观察 30 毫秒一次的自检），为了避免冲突，可能不得不为这些相似事件创建**单独的** `Base class` 派生类。

## 资源管理和线程安全 (Resource Management and Thread Safety)

- 传统的 C++ 实现中，如果主题没有正确地 `delete` 观察者，就会导致 **memory leak（内存泄漏）**。

- 这部分问题可以通过使用 **`shared_ptr`/`weak_ptr`** 来部分解决（安全地管理生命周期），但线程安全问题（如竞态条件 `race condition`、线程互斥 `thread contention` 等）仍然存在。

# 现代C++代替方案

现在的语言普遍通过更灵活的机制绕过了传统观察者模式的限制：

- **Java/C#:** 使用**匿名内部类**（Anonymous Inner Classes）、**`Closure`** 或 **`delegate`** 来实现回调机制。
- **C++ (Boost/STL):** 使用 **`Signal/Slots`** 或 **`function/bind`** 来实现

## SIGNAL/SLOT

**信号槽** 机制是一种在 **C++** 中实现 **对象间通信（解耦）** 的设计模式，尤其在 **GUI 编程**（如 Qt 框架）中被广泛应用。

- **信号 (Signal):** 当一个特定事件发生时，对象会 **发射 (emit)** 一个信号。信号本身不包含任何逻辑代码，它只是一个通知。
  
- **槽 (Slot):** 槽是一个普通的 **成员函数**，用于 **响应** 信号的发出。当信号连接到槽时，信号发射后，槽函数就会被自动调用。
## Signal/Slots 机制优势

它使用 **`std::shared_ptr`** 来管理**订阅者（观察者）**的生命周期，从而解决前面提到的 **`shared_ptr_ptr/weak_ptr`** 方案中的内存管理问题。

它可以完全**替换**传统的观察者模式，因为它不再需要观察者派生自任何基类。

它支持**任意**的成员函数签名作为槽（`Slot`），彻底解决了传统观察者模式中对函数**名字、参数、返回类型**的限制。

核心作用
- **解耦 (Decoupling):** 发出信号的对象（发送者）**不需要知道** 哪个或哪些对象（接收者）会响应它，也不需要知道这些响应的具体实现。它只管发射信号。
  
- **多对多连接:** 一个信号可以连接到多个槽，一个槽也可以接收来自多个信号的连接。
  
- **类型安全:** 信号槽的参数类型通常是严格检查的，确保连接的信号和槽具有兼容的参数列表。

优秀的 `Signal/Slots` 实现

- **`thread safe`** (线程安全)
- **`race condition free`** (无竞态条件)
- **`thread contention free`** (无线程互斥)

## 实现代码

```cpp

template <typename Signature>
class SignalTrivial;

/**
 * @brief SignalTrivial 类模板的特化版本，处理函数签名 RET(ARGS...)
 * @tparam RET 函数返回类型
 * @tparam ARGS 函数参数类型包
 * @example 
 *  * SignalTrivial<void(int)> signal;
 *  * signal.connect([](int x) { std::cout << x << std::endl; });
 *  * signal.call(42);
 */
template <typename RET, typename... ARGS>
class SignalTrivial<RET(ARGS...)>
{

public:
    using Functor = std::function<void (ARGS...)>;

    void connect(Functor&& slot)
    {
        // 使用std::forward完美转发
        slots_.emplace_back(std::forward<Functor>(slot));
    }

    void call(ARGS&&... args)
    {
        for (const Functor& slot : slots_)
        {
            slot(std::forward<ARGS>(args)...);
        }
    }
private:
    std::vector<Functor> slots_;
};
```
